Variablen beobachten

Lade folgendes Beispielprogramm:

  • In diesem Programm gibt es die beiden Variablen "variable_1" und "variable_2" die ständig ihren Wert ändern. Es gibt aber keine Ausgabe der Werte über die Zeichenfläche. Deswegen verwenden wir in dieser Übung den Chrome-Debugger, der uns die aktuellen Werte dieser Variablen anzeigen kann.

    Drücke auf der Tastatur die Taste "F12" und klicke im Debugger auf den Tab "Sources".

    Klicke oben links auf das Symbol "Show navigator".

  • Öffne die Ordner, bis das folgende Symbol mit einem Zufalls-Dateinamen angezeigt wird. Klicke auf den Zufallsdateinamen, so dass sich der Quellcode unseres Programms öffnet.

  • Klicke auf den Zufallsdateinamen, so dass sich der Quellcode unseres Programms öffnet.

  • Schließe den Datei-Navigator, indem du oben in der Mitte auf das Symbol "Hide Navigator" klickst und öffne den Debugger, indem du unten rechts auf das Symbol "Show debugger" klickst, falls der Debugger noch nicht angezeigt wird.

  • Markiere im Quellcode mit der Maus die Variable "variable_1" und klicke auf der Markierung die rechte Maustaste. Klicke mit der linken Maustaste auf "Add selected text to watches".

  • Unten rechts sieht man die beiden Variablennamen und jeweils die Meldung "not available".

  • Damit der Wert jeder Variablen zu einem bestimmten Zeitpunkt im Programmablauf angezeigt wird, fügt man einen Breakpoint in das Programm ein, an welchem das Programm pausieren soll.

    Dazu klickt man mit der linken Maustaste auf die Zeilennummer der Zeile, in welcher ein Breakpoint gesetzt werden soll.

  • Soabld das Programm an einem Breakpoint pausiert ist, kann man es Schritt für Schritt ausführen lassen. Dazu gibt es eine Steuerzeile mit Symbolen zur Programmsteuerung.

    Step: Mit dem Symbol "Step" wird die Anweisung ausgeführt und das Programm springt zur nächsten Anweisung. Wenn die nächste Anweisung in einer Funktion an einer anderen Stelle im Code steht, springt der Debugger zu der anderen Stelle.

  • "Step over next function call": Mit dem Symbol "Step over next function call" wird die Anweisung ausgeführt und wenn in der Anweisung ein Funktionsaufruf steht, dann wird dieser aufgerufen und ausgeführt, der Debugger geht aber in die nächste Zeile ohne in die aufgerufenen Funktion zu springen.

  • "Resume script execution": Mit dem Symbol "Resume script execution" werden alle Anweisungen ausgeführt, bis das Programm wieder beim markierten Breakpoint landet.

  • Wenn man versehentlich in eine andere Datei gesprungen ist, kann man jederzeit über einen Klick in der Tab-Leiste zum eigenen Programm zurückspringen.

  • Man kann auch mehrere Breakpoints in das Programm setzen und über das Symbol "Resume script execution" von Breakpoint zu Breakpoint springen.

    Im Fenster unten rechts kann man die aktuelle Werte der Variablen zu einem bestimmten Zeitpunkt beobachten.

  • Mit Hilfe des Debuggers, den es auch in vielen anderen Browsern gibt, kann man sich als ProgrammierIn das Programmierleben deutlich vereinfachen, indem man die Veränderungen von Variablen beobachtet und so logischen Programmier-Fehlern auf die Spur kommt.
